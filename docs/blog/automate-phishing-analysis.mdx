# How to Automate Phishing Email Analysis with Allama

Phishing remains the number one attack vector. According to Verizon's Data Breach Investigations Report, 36% of all breaches involve phishing. Yet most security teams still analyse phishing emails manually, spending 10-15 minutes per email on repetitive tasks.

In this tutorial, you'll build an automated phishing triage workflow that:
- Extracts indicators (URLs, attachments, sender information)
- Checks against multiple threat intelligence sources
- Auto-classifies severity based on findings
- Creates a case with all evidence
- Notifies analysts only when human review is needed

**Result: Reduce phishing triage time from 15 minutes to 30 seconds.**

## Prerequisites

Before starting, ensure you have:
- Allama running (see [installation guide](/quickstart/install))
- API keys for VirusTotal (free tier works)
- A Slack webhook URL (optional, for notifications)

## The Problem: Manual Phishing Analysis is Slow

Here's what a typical manual phishing analysis looks like:

1. **Receive report** — User forwards suspicious email (2 min)
2. **Extract sender** — Copy email address, check reputation (3 min)
3. **Extract URLs** — Manually copy each URL (2 min)
4. **Check URLs** — Submit to VirusTotal, URLScan one by one (5 min)
5. **Check attachments** — Download, submit to sandbox (5 min)
6. **Document findings** — Write up analysis in ticket (3 min)
7. **Respond** — Notify user, block indicators if malicious (2 min)

**Total: 15-20 minutes per email**

With 50+ phishing reports per day, that's over 12 hours of analyst time on repetitive work.

## What You'll Build

An automated workflow that handles steps 2-6 automatically:

```
Email Received → Extract Indicators → Check Threat Intel → Classify → Create Case → Notify (if needed)
```

## Step 1: Set Up Email Ingestion

First, configure how phishing reports reach Allama. You have several options:

### Option A: Webhook from Email Gateway

Most email security gateways (Proofpoint, Mimecast, Microsoft Defender) can forward reported emails to a webhook.

In Allama, create a new workflow with a **Webhook** trigger:

```yaml
trigger:
  type: webhook
  config:
    method: POST
```

### Option B: Mailbox Monitoring

Set up a dedicated mailbox (e.g., phishing@yourcompany.com) and use Allama's email integration to poll for new messages.

### Option C: Manual Submission

For testing, use Allama's manual trigger and paste email content directly.

## Step 2: Extract Indicators

Add an action to parse the email and extract indicators of compromise (IOCs):

```yaml
- ref: extract_indicators
  action: core.transform.reshape
  args:
    value:
      sender: ${{ trigger.data.from }}
      subject: ${{ trigger.data.subject }}
      urls: ${{ REGEX_FINDALL(trigger.data.body, 'https?://[^\s<>"]+') }}
      domains: ${{ REGEX_FINDALL(trigger.data.body, '(?:https?://)?([a-zA-Z0-9][-a-zA-Z0-9]*\.[a-zA-Z]{2,})') }}
      attachments: ${{ trigger.data.attachments }}
```

This extracts:
- Sender email address
- Email subject
- All URLs in the body
- All domains mentioned
- Any attachments

## Step 3: Check Against Threat Intelligence

Now, check each indicator against threat intelligence sources. Allama's scatter-gather pattern makes this easy:

```yaml
- ref: check_urls
  action: core.transform.foreach
  args:
    items: ${{ steps.extract_indicators.result.urls }}
    action: integrations.virustotal.scan_url
    args:
      url: ${{ item }}

- ref: check_sender_domain
  action: integrations.virustotal.domain_report
  args:
    domain: ${{ SPLIT(steps.extract_indicators.result.sender, '@')[1] }}

- ref: check_sender_reputation
  action: integrations.emailrep.check
  args:
    email: ${{ steps.extract_indicators.result.sender }}
```

These actions run in parallel, checking:
- Each URL against VirusTotal
- The sender's domain reputation
- The sender's email reputation

## Step 4: Auto-Classify Severity

Based on the threat intelligence results, automatically classify the email:

```yaml
- ref: classify
  action: core.transform.reshape
  args:
    value:
      is_malicious: ${{ 
        ANY(steps.check_urls.result, item.positives > 3) OR
        steps.check_sender_domain.result.positives > 5 OR
        steps.check_sender_reputation.result.suspicious == true
      }}
      severity: ${{
        IF(ANY(steps.check_urls.result, item.positives > 10), 'critical',
        IF(ANY(steps.check_urls.result, item.positives > 3), 'high',
        IF(steps.check_sender_reputation.result.suspicious, 'medium', 'low')))
      }}
      confidence: ${{
        IF(ANY(steps.check_urls.result, item.positives > 10), 'high',
        IF(ANY(steps.check_urls.result, item.positives > 3), 'medium', 'low'))
      }}
```

Classification logic:
- **Critical**: Any URL with 10+ VirusTotal detections
- **High**: Any URL with 3+ detections
- **Medium**: Suspicious sender reputation
- **Low**: No indicators found

## Step 5: Create Case with Evidence

Automatically create a case with all findings:

```yaml
- ref: create_case
  action: core.cases.create
  args:
    title: "Phishing Report: ${{ steps.extract_indicators.result.subject }}"
    severity: ${{ steps.classify.result.severity }}
    description: |
      ## Reported Email
      - **From**: ${{ steps.extract_indicators.result.sender }}
      - **Subject**: ${{ steps.extract_indicators.result.subject }}
      
      ## Indicators Found
      - **URLs**: ${{ LENGTH(steps.extract_indicators.result.urls) }}
      - **Domains**: ${{ LENGTH(steps.extract_indicators.result.domains) }}
      
      ## Threat Intelligence Results
      - **Classification**: ${{ steps.classify.result.severity }}
      - **Malicious**: ${{ steps.classify.result.is_malicious }}
      - **Confidence**: ${{ steps.classify.result.confidence }}
      
      ## Raw Results
      ```json
      ${{ JSON(steps.check_urls.result) }}
      ```
    tags:
      - phishing
      - automated-triage
```

## Step 6: Notify Analyst (If High Severity)

Only alert analysts when human review is needed:

```yaml
- ref: notify_analyst
  action: integrations.slack.post_message
  condition: ${{ steps.classify.result.severity IN ['critical', 'high'] }}
  args:
    channel: "#security-alerts"
    text: |
      🚨 *High-Severity Phishing Detected*
      
      *Subject*: ${{ steps.extract_indicators.result.subject }}
      *From*: ${{ steps.extract_indicators.result.sender }}
      *Severity*: ${{ steps.classify.result.severity }}
      *Confidence*: ${{ steps.classify.result.confidence }}
      
      <${{ steps.create_case.result.url }}|View Case>
```

## Complete Workflow YAML

Here's the complete workflow you can import directly into Allama:

```yaml
type: workflow
definition:
  title: Automated Phishing Triage
  description: Automatically analyse reported phishing emails
  
  trigger:
    type: webhook
    config:
      method: POST
  
  actions:
    - ref: extract_indicators
      action: core.transform.reshape
      args:
        value:
          sender: ${{ trigger.data.from }}
          subject: ${{ trigger.data.subject }}
          urls: ${{ REGEX_FINDALL(trigger.data.body, 'https?://[^\s<>"]+') }}
          domains: ${{ REGEX_FINDALL(trigger.data.body, '(?:https?://)?([a-zA-Z0-9][-a-zA-Z0-9]*\.[a-zA-Z]{2,})') }}
    
    - ref: check_urls
      action: core.transform.foreach
      args:
        items: ${{ steps.extract_indicators.result.urls }}
        action: integrations.virustotal.scan_url
        args:
          url: ${{ item }}
    
    - ref: check_sender_domain
      action: integrations.virustotal.domain_report
      args:
        domain: ${{ SPLIT(steps.extract_indicators.result.sender, '@')[1] }}
    
    - ref: classify
      action: core.transform.reshape
      args:
        value:
          is_malicious: ${{ ANY(steps.check_urls.result, item.positives > 3) }}
          severity: ${{ IF(ANY(steps.check_urls.result, item.positives > 10), 'critical', IF(ANY(steps.check_urls.result, item.positives > 3), 'high', 'medium')) }}
    
    - ref: create_case
      action: core.cases.create
      args:
        title: "Phishing: ${{ steps.extract_indicators.result.subject }}"
        severity: ${{ steps.classify.result.severity }}
        tags: [phishing, automated]
    
    - ref: notify
      action: integrations.slack.post_message
      condition: ${{ steps.classify.result.severity IN ['critical', 'high'] }}
      args:
        channel: "#security-alerts"
        text: "🚨 High-severity phishing detected. Case: ${{ steps.create_case.result.url }}"
```
